ref和reactive的差别

ref---》基本数据类型和
reactive----》对象类型类型

使用场景

若需要一个基本类型的响应式类型，必须要ref
若需要一个响应式对象类型，层次不深，ref，reactive都可以
若需要一个响应式对象类型，且层次较深，推荐使用reactive；

ref

ref针对的是基础数据类型

ref 将变量变成可以浏览器实时渲染，等于获取网页文档标签元素对象，对对象的值进行修改

import ref from 'vue'
let people = ref('zhansan')

ref函数将people从字符串变成对象
在后续的对people的使用，要用people.value对值的获取
但在html里面变量直接使用people对象。
这个人名字：{{people}} ref创建的变量改变值必须需要value进行修改
直接修改值也还是响应式数据，并没有修改地址，只是修改ref对象里面的值

rective

将对象变量(数组也是)变成可以浏览器实时渲染的函数
let people = reactive({
id : 123,
name : 'zhangsan'
}) reactive 修改值会重新分配一个对象，会失去响应式（可以使用Object.assign进行替换）
Object.assihn(people,people1)
不能直接修改值 people = reactive (people1) people = people1
